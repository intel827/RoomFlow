# 프로젝트 이름: RoomFlow

## 1. 프로젝트 개요
- 목적: 사내 회의실 예약
- 대상 사용자: 사내 임직원
- 핵심 기능 요약: 사내 회의실을 예약하고 부서/팀간 회의가 겹치지 않도록 관리한다.

## 2. 기술 스택
- 프론트엔드: React
- 백엔드: Node.js
- 데이터베이스: sqlite
- 배포 환경: 상관없음

## 3. 기능 요구사항
### 3.1 사용자 기능
- [v] 회의실 목록 조회
- [v] 날짜/시간 선택
- [v] 예약 생성
- [v] 예약 수정
- [v] 예약 삭제
- [v] 중복 예약 방지
- [v] 현재 회의실의 상태 확인
- [v] 취소된 예약인 경우 사유 확인

### 3.2 관리자 기능
- [v] 회의실 추가/삭제
- [v] 예약 강제 취소확인
- [v] 통계 조회

## 4. UI/UX 구조
### 4.1 페이지 목록
- 메인 페이지 : 회의실 화면에 띄워줌(현재 상태 보여줌), 우클릭으로 예약 추가/변경/삭제
- 회의실 상세 페이지 : 해당 회의실의 이 시간 이후의 예약 목록
- 관리자 기능 : 회의실 추가/삭제, 회의실 별 예약 취소 기능(사유 작성)

### 4.2 화면 흐름
1. 사용자 → 메인 페이지
2. 회의실이 비주얼적으로 보이고 현재의 상태를 알 수 있음
3. 좌클릭하면 상세나오고 우클릭시 예약 추가/예약 수정/예약 삭제 가능
4. 관리자도 같은 화면이지만 우클릭에 관리자만의 기능이 추가된다

## 5. API 명세
### GET /rooms
- 설명: 회의실 목록 조회
- 응답 예시: JSON 형태로 회의실 이름과 상태로 이루어진 목록을 조회한다.

### GET /room/${roomId}
- 설명: 회의실의 기본정보와 현재 시각 이후의 예약 목록을 조회한다.

### POST /reservations
- 설명: 회의실을 예약한다
- 요청 바디: 미정
- 응답: 미정

### PUT /reservations
- 설명: 회의실 예약을 수정한다

### DELETE /reservations
- 설명: 회의실 예약을 삭제한다

## 6. 비즈니스 규칙
- 예약 시간은 30분 단위
- 중복 예약 불가
- 현재 시각 이전으로는 예약 불가 (과거 시간 예약 차단, alert으로 안내)
- 관리자만 회의실 추가 가능
- 관리자만 예약 취소 가능, 사유 필수로 작성
- 관리자가 미래 예약이 있는 회의실 삭제 시 경고 확인 (confirm) 후 삭제 진행
